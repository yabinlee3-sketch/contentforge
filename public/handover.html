### 2.5 paywall.ts 验证逻辑（重要，新 AI 必须理解）
当前是客户端验证（localStorage），不是服务端验证。

**代码逻辑：**
1. 用户打开页面 → `isPro()` 检查 localStorage 中的 `cf_pro_expiry`
2. 如果有值且未过期 → 返回 `true`（Pro 激活）
3. 如果没有或已过期 → 返回 `false`（免费用户）
4. 免费用户显示剩余次数（`remainingFree()`，也是 localStorage 计数）
5. 用户点击"Upgrade Now" → `openCheckout()` 打开 LemonSqueezy 结账弹窗
6. 用户付完款 → 弹窗关闭 → 触发了 `postMessage` 事件监听
7. `initCheckoutListener()` 捕获到 checkout 完成 → 设置 30 天有效期
8. 页面刷新后保持 Pro 状态

**`activatePro()` 函数：** 设置 30 天后的过期时间戳到 localStorage
**`checkRecentPurchase()` 函数：** 页面加载时检查是否有新支付（备用方案）
**`clearProStatus()` 函数：** 清除 Pro 状态（降级用）

**这个验证方式的风险：** 用户可以通过 Chrome DevTools 修改 localStorage 绕过。
解决方案（等 boss 注册 Supabase 后启用）：把验证改为 API 调用 + 数据库存储。
